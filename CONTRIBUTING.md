# Contributing Guidelines

Thank you for your interest in contributing to termux-workspace-sync!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/termux-workspace-sync.git`
3. Create a feature branch: `git checkout -b feature/amazing-feature`
4. Make your changes
5. Test thoroughly on Termux
6. Commit: `git commit -m 'Add amazing feature'`
7. Push: `git push origin feature/amazing-feature`
8. Submit a Pull Request

## Code Standards

- Follow bash best practices
- Write clear, documented code
- Test on actual Termux environment
- Keep scripts portable
- Handle errors gracefully

## Commit Message Format

```
type(scope): subject

body (optional)

footer (optional)
```

**Types:** feat, fix, docs, style, refactor, test, chore

**Examples:**
- `feat(sync): add multi-repo sync support`
- `fix(auth): resolve SSH key detection`
- `docs(readme): update installation steps`

## Testing

- Test all changes on Termux
- Verify sync functionality
- Check error handling
- Test on slow networks

## Pull Request Process

1. Update documentation
2. Add usage examples
3. Test on Termux
4. Request review
5. Address feedback

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Provide constructive feedback
- Follow project guidelines
